    字符集是什么

    我们知道计算机底层的数据都是 1 或 0 二进制的 例如 00000000 就是一个字节 每个位是一个bit

    # bit 字节 字符 字符集 编码方式 都是什么关系？

    8个bit = 1个字节

英文字母字符 = 1个字节 

汉字字符 = 2个字符/3个字符

    我们知道计算机底层的数据都是 1 或 0 二进制的 例如 00000000 就是一个字节 每个位是一个bit

    00000000 到 11111111 是 十进制 0-255 也就是256个状态

    对于英文编码来说 26个字母加上标点符号 256个状态是足够使用了的，但是对于中文，博大精深的汉字还说，远远不够

    使用到的位置就是在255+了 也就是会出现2个字符或3个字符了

    # 什么是字符集？

    它就是一个表，是一个 二进制与字符的对照表，二进制就是字符的地址
    1

    汉字编码简明对照表

    例如：啊 1601 这里的1601是十进制的 我们需要换算成二进制 也就是 11001000001

    我们看二进制的位数是11位 需要占用2个字节，补全2个字节后二进制是 00000110 01000001 也有的汉字会占用3个字节。

    共同存储时 占用2个字节的汉字 对应的就会比3个字节的字符的地址前就会多出8个0来补位，存储的大小就会多出几倍

    为了更好的减少存储大小，这时出现了编码方式，来解决这一问题

    你可以认为成，加了一个约定的算法，减少了多个0的出现，减少了存储大小

    # 它是如何减少的存储大小呢？

    单字节 我们就用0前缀后面的7位是二进制 例如 0xxxxxxx

    多字节 则是以1前缀 有几个1就是代表是几个字节

    例如 110xxxxx 10xxxxxx 从左往右，有2个1，是两个字节，xxxxx 加上后边的xxxxxx 则为当前字符的二进制地址

    编码方式就是对字符二进制处理的压缩算法 - 个人理解

    # Unicode 和 utf-8

    Unicode 是字符集 （国际通用字符集）

    utf-8 是编码方式

    关于字符集的还有很多 GBK GB2312 等等 但是对于开发来说utf-8 应该够用了
